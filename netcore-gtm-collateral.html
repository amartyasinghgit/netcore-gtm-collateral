Integration Steps

Setup in 4 steps

Follow these steps to connect your website to Netcore Cloud Customer Engagement via GTM.

1
Google Tag Manager

Set Up Your GTM Account & Container

Create or access your GTM account and set up a container for the target website. Each container gets a unique GTM ID (e.g. GTM-XXXXXXX) which is embedded once in your website's <head> and <body>.

Tip: Use a single GTM container per domain. Ensure the GTM snippet is placed as high in the <head> as possible for reliable tag firing.

2
Netcore CE Dashboard

Create an Asset on the Netcore CE Panel

Contact the Netcore integration team to provision your asset, or follow the Web SDK integration guide to self-serve. This generates your unique Panel Identifier and Website Identifier — both required for the JS snippet.

Note: Once you access the JS file from the Netcore CE panel, your Panel Identifier and Website Identifier will be pre-populated in the code.

3
GTM → Tags

Copy the SDK Code & Create the Base Tag

In Netcore CE, click "Get Javascript SDK", copy the snippet, and create a new Custom HTML tag in GTM. Prefix the tag name with Netcore for easy organization. This is your Base Tag — the foundation for all subsequent tracking.

HTML — Base Tag
<!-- Place in Custom HTML Tag, trigger: All Pages -->
<script src='//cdnt.netcoresmartech.com/smartechclient.js'></script>
<script>
  smartech('create', '<Smartech Panel Identifier>');
  smartech('register', '<Website Identifier>');
  smartech('identify', '<User Primary Key — email, mobile, or ID>');
</script>
01

create

Initializes SDK with your unique Panel Identifier

02

register

Associates the session with your Website Identifier

03

identify

Maps the session to a user's primary key. Leave blank for anonymous tracking

04

trigger

Set this tag to fire on All Pages so it loads universally

4
GTM → Variables, Triggers & Tags

Build Variables, Triggers & Event Tags

Define the data you need to capture, set the conditions for tag firing, and deploy tags for each tracking use case. The following sections detail the configuration for each layer.

Layer 1 — Variables

Capture dynamic user data

Variables extract values from the page at runtime. Define them once and reference them across all tags using double-brace syntax: {{Variable_Name}}

✉️
Email ID
Custom JavaScript Variable

Dynamically captures the email entered in any text field across the site — login, registration, newsletter, etc.

JavaScript
function() {
  try {
    var searchEmail = function(input) {
      for (i=0; i<input.length; i++) {
        email = input[i].value;
        if(email.match(
          /^([\w-]+...@...\....)$/i
        )){ return email; }
      }
    }
    return searchEmail({{Form Element}});
  } catch(e) { return ""; }
}
📱
Mobile Number
Custom JavaScript Variable

Captures 10-digit Indian mobile numbers entered anywhere across the website using regex-based matching.

JavaScript
function() {
  try {
    var searchMobile = function(input){
      for(i=0; i<input.length; i++){
        mobile = input[i].value;
        if(mobile.match(
          /^(\+91[\-\s]?)?[0]?(91)?[789]\d{9}$/i
        )){ return mobile; }
      }
    }
    return searchMobile({{Form Element}});
  } catch(e) { return ""; }
}
👤
User Name / Demographics
DOM Element Variable

Registration-specific fields (name, age, gender) are captured via their DOM element ID or CSS Selector — inspect page source to identify the right selector.

Use Element ID when available. Fall back to CSS Selector if no ID is assigned to the element.

🔘
Radio Button Selection
Click ID / Form ID Variable

Captures the user's selection from radio button groups (e.g., gender). Configured using Click ID or Form ID based on your website's implementation.

Enable all GTM built-in click variables before configuring radio-button click tracking.

Layer 2 — Triggers

Define when tags fire

Triggers are the conditions that determine when a tag is deployed. Each event tag needs at least one trigger.

Built-in

All Pages

GTM's default trigger. Fires on every page load. Used for the Base Tag and Page Browse Tag.

Custom

Clickable Links

Fires when a user clicks any internal or external hyperlink. Use Click → Just Links type. Enable all click variables first.

Custom

All Elements Click

Tracks clicks on any specified page element (buttons, banners, CTAs) with custom filter conditions.

Custom

Form Submission

Fires on contact form, registration, or any data entry form submit. Ideal for capturing lead data.

Layer 3 — Event Tags

Deploy tracking for every use case

All Netcore CE tracking tags use the Custom HTML tag type in GTM. Below is the complete reference for standard use cases.

Tag Name Use Case Trigger Notes
Netcore CE Base Tag
 SDK initialization; connects website to Netcore CE panel All Pages Must be the first tag. Includes create, register, identify calls.
Netcore CE Page Browse
 Tracks site navigation & time spent per page All Pages Set tag sequencing to fire after Base Tag.
Netcore CE User Registration
 Captures new customer sign-up data to a CE list Form Submit Pass List ID and form activity. Uses {{Email_Input}}, {{Mobile_Input}}.
Netcore CE Add To Cart

 E-commerce: product removed from cart event Element Click Mirror of Add To Cart with remove action flag.

 E-commerce: checkout initiated event Element Click Capture cart value and item count at time of checkout.

 Tracks how far users scroll down each page Custom Event Import a scroll-tracking recipe container from the GTM community.

 Captures e-newsletter subscription data Form Submit Uses {{Email_Input}}. Feeds into newsletter list for email campaigns.
Deploy & Validate

Test before you go live

GTM's built-in Preview & Debug mode lets you validate every tag, trigger, and variable in your own browser before publishing to all users.

  • Enter Preview Mode in GTM — only your browser runs the debug container.
  • GTM highlights any tag or trigger errors before you can commit the container.
  • Verify tokens appear on your Netcore CE Dashboard after browsing tracked pages.
  • Test each stage independently — after variables, after triggers, and after each individual tag.
  • Publish the container once all checks pass. All tags are now live for all users.
  • Use version numbers for every publish. Roll back instantly if issues are found in production.

Platform Notes: For Shopify sites, only event tracking can be configured via GTM. For Magento sites, both event tracking and BPN (Browser Push Notifications) can be configured via GTM.

SDK Size: The JavaScript file is 150 KB on the Netcore server and 60–65 KB when served via CDN. Once integrated, you'll see tokens arriving on your Netcore CE dashboard immediately.
